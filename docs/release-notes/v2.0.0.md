## Highlights

- **OAuth2** endpoints implemented: client_credentials, refresh_token, authorization_code.
  - Token persistence adapters: InMemory (default), Sled (disk), Redis (optional).
- **Offline & interactive API docs**:
  - Optional vendored `swagger-ui` distribution available in `crates/api/static/swagger-ui/` (use `scripts/fetch-swagger-ui.*` to vendor the bundle).
  - Local interactive UI (`crates/api/static/swagger-ui/interactive.js`) with Try-it-out (GET/POST/PUT), **Auth** panel (client credentials fetch + clear token), parameter forms (query + form) and OAuth helpers (`fetchTokenClientCredentials`, `performAuthorizationCodeFlow`).
- **Testing & CI**:
  - Unit, integration and e2e tests (including simulated Authorization Code flow and optional Redis integration tests).
  - CI workflow to lint OpenAPI with Spectral and run `cargo test -p verseguy-api` (`.github/workflows/docs-ci.yml`).

## Usage notes

- Run the service and open the docs:

```bash
cargo run -p verseguy-api
# then open http://localhost:3000/docs
```

- Client Credentials demo (in browser console):

```js
window.fetchTokenClientCredentials('demo', 'secret').then(console.log).catch(console.error)
```

- Authorization Code demo (opens popup + exchanges code):

```js
window.performAuthorizationCodeFlow({ clientId: 'demo', clientSecret: 'secret' }).then(console.log).catch(console.error)
```
